Resources & Reference Architecture

Why LLMs Get Expensive in Production

And how a control-plane architecture fixes it. The cost isn't the model — it's the non-deterministic loops, ungoverned traffic, and missing observability around it. The fix: push deterministic work into workflows, expose them as tools, and route everything through a gateway that enforces limits.

Every resource below maps to one part of that fix. Tap a card to open it.

Step 1 Make the Work Deterministic
CLI · Arazzo → MCP
Arazzo Workflows, Wrapped as an MCP Tool
Design business tasks as deterministic Arazzo workflows — no tokens, no model reasoning — then expose them to the LLM as a single MCP tool. The expensive agent loop calls one tool instead of improvising the whole sequence.
wso2.com/api-platform/docs/tools/arazzo-mcp-gen-cli/quick-start-guide
Step 2 Govern & Route the Traffic
Data Plane
WSO2 AI Gateway
Rate limits, token budgets, and routing at the edge — where runaway LLM cost is actually contained.

 Control Plane
WSO2 AI Workspace
